## 一、项目概况

SmartCode 是一个终端 AI 编码 Agent（类似 Claude Code 的开源 Python 实现），
核心是一个 think/act/verify 循环引擎，支持多工具调度、会话持久化、文件回退等。

| 维度 | 数据 |
|:-----|:------|
| 源码量 | ~42K 行，102 个 .py 文件 |
| 测试量 | ~11K 行，31 个测试文件 |
| 运行时依赖 | **零**（纯 Python 标准库） |
| 入口 | `pip install -e .` 后 `minicode` 命令 |

## 三、审查系统（本次核心产出）

### 3.1 两级审查

```
宽松审查（毫秒级）
  ├── 正则：硬编码密钥、SQL 注入、eval 滥用、TODO 遗留
  └── AST ：bare except 检测
  → 命中 critical → 阻断写入（force=true 逃生）
  → 命中小问题 → 放行 + 提示

严格审查（秒级）
  ├── review agent：跨文件影响分析 + 代码质量
  ├── test agent：Docker 沙箱跑测试
  └── 触发条件：安全路径 / diff 特征 / 历史问题率 / 新人代码
```

### 3.2 插拔式设计

```
review/ 目录存在 → 审查系统自动接入
review/ 目录删除 → 完全回归原版（try/except ImportError 兜底）
改写的文件只有 10 个，约 372 行净代码
```

### 3.3 运行时模式切换

```
MINICODE_REVIEW_MODE=off     对照组，不审查
MINICODE_REVIEW_MODE=loose   实验组1，只开宽松审查
MINICODE_REVIEW_MODE=strict  实验组2，全开 + 子 Agent

off → strict 切换 → 自动触发全面回补扫描
```

### 6.2 编码任务测试

| 模式 | 正确数 | 总数 | 正确率 | 总耗时 | 平均每任务 |
|:-----|:------:|:----:|:------:|:------:|:----------:|

> 同时提供了额外的安全保障。
