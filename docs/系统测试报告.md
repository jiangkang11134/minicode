## 三、宽松审查测试

| 样本 | 规则 | 预期 | 实际 | 耗时 | 状态 |
|:-----|:-----|:----:|:----:|:----:|:----:|
| `api_key = "sk-xxx"` | hardcoded-secret | 阻断 | 检出 | 2.6ms | ✅ |
| `execute(f"SELECT...{name}")` | sqli | 阻断 | 检出 | <1ms | ✅ |
| `eval(user_input)` | unsafe-eval | 阻断 | 检出 | 0.6ms | ✅ |
| `try/except:` | bare-except | 阻断 | 检出（正则补充） | <1ms | ✅ |
| `# TODO: add validation` | todo-left | 提示 | 检出 | <1ms | ✅ |
| `def add(a,b): return a+b` | (clean) | 放行 | 放行 | <1ms | ✅ |

**分析**：bare-except 漏检是因为单行 `except:` 需要前面有 `try:` 上下文才能通过 AST 检测。
已补充正则 `\bexcept\b` 检测作为降级方案，单行 `except:` 也能覆盖。
**修复后拦截率：100%。**

**综合指标**：
- 准确率：100%（没有误报）
- 拦截率：100%（6/6，正则补充后全覆盖）

## 六、三级模式对比

| 模式 | 功能 | 适用场景 |
|:-----|:------|:---------|
| **off** | 完全不审查，纯编码 | 快速原型、信任的开发者 |
| **loose** | 正则+AST 检查，阻断密钥泄露 | 日常开发，防止低级错误 |
| **strict** | 全部功能 + 子 Agent 审查 + Docker 沙箱测试 | 安全敏感代码，多人协作 |

环境变量切换：
```bash
export MINICODE_REVIEW_MODE=strict   # 开启全部审查
export MINICODE_REVIEW_MODE=loose    # 只开宽松
export MINICODE_REVIEW_MODE=off      # 关闭
```
