# SmartCode Python 学习路线图

> 本文档回答一个问题：**哪些东西需要花时间、哪些看一眼就行**
>
> 分类标准：
> - 🔴 **必须看懂代码** — 面试必问，需要能讲清楚实现细节
> - 🟡 **理解设计意图** — 知道为什么要有、怎么实现的，不需要读代码
> - 🔵 **知道有它就行** — 只需要知道它的存在和一句话定位
---

## 一、🔴 必须看懂代码（6 个文件）

这些是 SmartCode 的核心骨架，面试官 90% 的问题会围绕它们展开。
**需要：读懂每行代码、能画出流程图、能回答"为什么这么设计"。**

### 1. `main.py` — 入口与启动流程

| 维度 | 内容 |
|------|------|
| 核心逻辑 | 参数解析 → 路由到纯 CLI 命令 → 初始化所有子系统 → 双路径分叉 |
| 关键函数 | `main()` 的全部阶段 |
| 必须理解 | `parse_known_args` vs `parse_args`、多级 return/SystemExit 路由、load_runtime_config 失败处理 |
| 面试可能问 | "启动流程是怎么样的""配置加载失败会怎么样""为什么用 parse_known_args" |

### 2. `agent_loop.py` — Agent 核心循环

| 维度 | 内容 |
|------|------|
| 核心逻辑 | prelude → step 循环（model→tool→verify）→ coda |
| 关键函数 | `run_agent_turn()`、`_execute_single_tool()`、`_model_next()` |
| 必须理解 | step 循环的完整流程、串行与并行工具执行的区别、异常处理链路 |
| 面试可能问 | "agent 的 think/act 循环怎么转的""工具执行超时怎么处理""模型调用失败怎么办" |

### 3. `turn_kernel.py` — 单轮状态机

| 维度 | 内容 |
|------|------|
| 核心逻辑 | 每一步的策略推导、模型返回后的决策分发、停止条件判断 |
| 关键函数 | `decide_assistant_turn()`、`decide_tool_turn()`、`derive_turn_step_policy()` |
| 必须理解 | 4 种决策结果（progress/retry/fallback/final）的区别和触发条件 |
| 面试可能问 | "模型返回空响应怎么处理""什么时候会触发 widening""verification gate 是什么" |

### 4. `session.py` — 会话持久化与恢复

| 维度 | 内容 |
|------|------|
| 核心逻辑 | 会话的持久化/加载、checkpoint 创建/回退、增量 delta 策略 |
| 关键数据结构 | `SessionMetadata`、`FileCheckpoint`、`SessionData` |
| 必须理解 | checkpoint 的生命周期、rewind 的执行流程、增量保存为什么用 delta |
| 面试可能问 | "rewind 怎么恢复文件的""增量 delta 怎么合并的""AutosaveManager 怎么工作的" |

### 5. `memory.py` — 三层记忆系统

| 维度 | 内容 |
|------|------|
| 核心逻辑 | 三层 scope（user/project/local）、BM25 检索、记忆注入 |
| 关键类 | `MemoryManager`、`MemoryEntry` |
| 必须理解 | 三层的区别和生命周期、`get_relevant_context()` 的检索流程 |
| 面试可能问 | "三层记忆分别存什么""BM25 怎么算相关性的""压缩时记忆怎么保护" |

### 6. 一个完整工具（如 `tools/read_file.py`）

| 维度 | 内容 |
|------|------|
| 核心逻辑 | 工具定义、参数校验、执行函数、ToolDefinition 注册 |
| 必须理解 | `_validate` 和 `_run` 的分工、`ToolDefinition` 的字段含义 |
| 面试可能问 | "一个工具从注册到执行完整链路是什么""参数校验失败怎么办" |

## 二、🟡 理解设计意图（15 个模块）

这些是重要但不紧急的。**不需要读代码，但需要能用 2-3 句话说清楚"为什么要有""怎么实现的"。**

### 核心支撑

| 模块 | 一句话定位 | 需要理解的设计意图 |
|------|-----------|------------------|
| `config.py` | 加载运行时配置（API key、模型、MCP） | 配置优先级：环境变量 > settings.json > 默认值 |
| `tooling.py` | 工具注册表，维护 name→function 映射 | ToolRegistry 内部是字典 + ToolDefinition，execute 就是查字典+调用 |
| `model_registry.py` | 模型适配器工厂，根据模型名创建对应适配器 | 自动检测 provider（Anthropic/OpenAI），创建对应的 adapter |
| `model_switcher.py` | 运行时热切换模型 | 主模型挂了自动切 fallback，按阶梯降级 |
| `context_manager.py` | token 估算 + 上下文窗口管理 | CJK 感知的 token 计数、判断上下文是否快满了 |
| `context_compactor.py` | 上下文压缩，多策略渐进式 | 4 层防御：微压缩→可恢复压缩→主动压缩→反应式压缩 |
| `prompt.py` | 组装系统提示 | 把 skills/MCP/记忆/配置 拼成一条 system prompt |
| `permissions.py` | 路径/命令/编辑审批 | TTY 模式问用户、非 TTY 模式自动判断风险 |
| `types.py` | 核心类型定义 | `ChatMessage`、`AgentStep`、`ModelAdapter` 等核心接口 |
| `state.py` | 全局状态机（类似 Redux store） | 统一管理 session_id/busy/tool_calls，所有模块从这读 |
| `tty_app.py` | 全屏 TUI 入口 | 事件驱动、alt-screen、节流渲染 |
| `headless.py` | 非交互模式（管道/CI/CD） | 逐行读 stdin，按优先级路由 |

### 工具文件（选 2-3 个理解即可）

| 模块 | 一句话定位 |
|------|-----------|
| `tools/write_file.py` | 写入文件，自动创建目录 |
| `tools/edit_file.py` | 精确替换文件中的某段文本 |
| `tools/run_command.py` | 执行 shell 命令，含安全过滤 |
| `tools/grep_files.py` | 搜索文件内容 |
| `tools/task.py` | 派生子 agent 执行复杂任务 |
| `tools/web_search.py` | 搜索网络 |
| `tools/web_fetch.py` | 获取网页内容 |
| `tools/git.py` | Git 操作 |

### TUI 核心层

| 模块 | 一句话定位 |
|------|-----------|
| `tui/input_handler.py` | 输入分发：命令→快捷工具→agent 后台运行 |
| `tui/renderer.py` | 整屏渲染 + 增量 hash 缓存 |
| `tui/transcript.py` | 会话历史渲染 + 视觉换行 |

## 四、推荐学习路线

### 第一阶段（3-5 天）：核心 6 文件

```
main.py → agent_loop.py → turn_kernel.py → session.py → memory.py → tools/read_file.py
```

每读完一个，试着用 2-3 句话说清楚：**这个文件解决了什么问题、核心流程是什么、关键设计决策是什么。**

### 第二阶段（2-3 天）：理解设计意图

把🟡里的 15 个模块挨个看一遍 docstring + 类/函数签名，不看实现。目标是：**面试官提到任何一个，你都能说出它是干什么的、为什么需要它。**

### 第三阶段（1 天）：准备面试话术

选 3 个你最熟悉的核心模块，准备：

1. **一句话定位**："session.py 是会话持久化模块，负责 checkpoint 和 rewind"
2. **核心流程图**：画出数据流
3. **一个设计决策**："为什么用 delta 增量保存不用全量？因为..."
4. **一个你改进的点**：比如"我在读代码时发现 Windows 下 BOM 编码问题，提了一个 PR 修复"

## 六、面试高频问题预测

### 必问

- "这个项目的整体架构是什么样的？"
- "Agent 循环是怎么工作的？"
- "记忆系统怎么实现的？"
- "如何保证上下文不溢出？"
- "模型调用失败怎么处理？"

### 大概率问

- "Session 的 checkpoint 和 rewind 怎么实现的？"
- "工具注册和执行的完整链路？"
- "这个项目跟 Claude Code / LangChain 有什么区别？"
- "你在项目里做了什么贡献？"

### 小概率问

- "控制论那套是怎么回事？"（一句话带过：可选扩展层，用 PID 控制上下文压缩）
- "TUI 是怎么渲染的？"
- "permissions 怎么做的？"
