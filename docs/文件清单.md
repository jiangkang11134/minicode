### 审查系统架构

```
agent_loop 三钩子
    ├── 钩子1 on_before_write()  → hooks._pre_review_content()  → 阻断/放行
    ├── 钩子2 on_file_written()  → mode_engine.should_trigger_strict()
    │       ├── True  → _spawn_review_sub_agent()  → task(review)
    │       │            └── PASS → _spawn_test_sub_agent() → sandbox_test
    │       └── False → 仅更新 import map
    └── 钩子3 on_turn_end()      → promote_findings() → promotion.py
                ├── 重要 → skill / 全局记忆
                └── 不重要 → memory.py (review-findings.json)
```

### 三模诊断模式

| 模式 | on_before_write | on_file_written | on_turn_end |
|:----:|:---------------:|:---------------:|:-----------:|
| **off** | 放行（no-op） | 跳过 | 跳过 |
| **loose** | 阻断 critical 写入 | 更新 import map | 注入发现 |
| **strict** | 阻断 critical 写入 | 更新 import map + 审查子 Agent | 注入发现 + 沉淀 |
